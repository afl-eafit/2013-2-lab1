Programming lab 1: Testing emptiness of automata

Submission deadline: Friday, August 16 (11:59 p.m.)

Introduction

The aim of this programming assignment is to write a Haskell program that given a non-deterministic finite automaton NFA, determine if L(NFA) = ∅.

Assignment (80%)

  • Download the GitHub repository agomezl/Empty.
    • The module EAFIT.CB0081.Data.NFA implements a non-deterministic finite automaton.
    • The module EAFIT.CB0081.Data.NFA.Examples gives an example of the data type NFA.
  • (70%) Using the module EAFIT.CB0081.Data.NFA, define a function. 
    	  empty :: NFA state symbol -> Bool
    that test whether L(NFA) = ∅.
    Using the module EAFIT.CB0081.Data.NFA.Examples, for example, 
    GHCi> empty emptyNFA
True
GHCi> empty nonemptyNFA
False
  • (10%) Annotate your source code in a way that explains your solution for the assignment. Improve it with HLint .

Clean code (10%)

Before submitting your code, clean it up! Clean code:
  • Does not have long lines (at most 80 columns).
  • Has a consistent layout.
  • Has type signatures for everything.
  • Has good comments.
  • Has no junk (unused code, commented code, unnecessary code).
  • Has no overly complicated function definitions.
  • Does not contain any repetitive code.

Submission (10%)

Submission is electronic. Send an email to asicard(at)eafit(dot)edu(dot)co and agomezl(at)eafit(dot)edu(dot)co and attach a compressed file (.zip or .tar.gz) with your solution.

Your submission has to include the following:

  • EmptyNFA.hs, a Haskell module file with your solution for the assignment.
  • README (or README.md), a file containing at least the following information.
    • The name of your program.
    • Your name(s).
    • A general description of your program.
    • Information on how to use your program.
